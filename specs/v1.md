# DocExtract — Spec

> Living document. Update as the project evolves.

---

## 1. Overview

A local-first desktop application for structured data extraction from PDF documents. Users define reusable extraction templates (JSON schemas), upload PDFs, and receive structured JSON output — with optional CSV/Excel download for flat schemas.

**Stack**
- Frontend: Electron + React + shadcn/ui
- Backend: Python service (FastAPI or Flask)
- Extraction engine: [`doc-intelligence`](https://github.com/zeel-04/doc-intelligence) (OpenAI-backed, Pydantic models)
- IPC: Electron ↔ Python via HTTP (localhost) or child process

---

## 2. Core Concepts

### Template
A named, reusable extraction configuration. Contains:
- A user-defined name and optional description
- A JSON schema defining the fields to extract (mapped to a Pydantic model at runtime)
- Optional extraction settings (pages to process, extraction mode, citation toggle)

### Extraction Job
A single run: one PDF + one template → one JSON output. Ephemeral by default (not persisted unless the user explicitly saves or exports).

### Flat Schema
A schema is considered flat if all fields are top-level primitives (no nested objects or arrays). Only flat-schema results offer CSV/Excel download.

---

## 3. Features

### 3.1 Template Management

- **Create** a template: give it a name and define a JSON schema via a form or raw JSON editor
- **Edit** an existing template
- **Delete** a template
- **List** all saved templates
- Templates are persisted locally (JSON file or SQLite)

JSON schema input must be validated before saving. Field types supported: `string`, `number`, `boolean`, `integer`. Nested objects and arrays are allowed but will disable flat export.

### 3.2 Document Extraction

1. User selects a template from the list
2. User uploads a PDF (digital or scanned)
3. App sends the PDF + schema to the Python backend
4. Backend uses `doc-intelligence` to run extraction
5. Result is displayed as formatted JSON in the UI

**PDF types supported:**
- Digital PDF (local file path passed to `DocumentProcessor.from_digital_pdf`)
- Scanned PDF (if `doc-intelligence` supports OCR path — to be confirmed)

**Extraction config (per template, optional overrides):**
- `extraction_mode`: `single_pass` (default) or other modes the library exposes
- `page_numbers`: optionally restrict to specific pages
- `include_citations`: toggle citation metadata in output

### 3.3 JSON Output View

- Display extracted JSON in a readable, syntax-highlighted viewer
- Show raw `extracted_data` prominently
- Optionally show `metadata` (citations) in a collapsible section
- Copy-to-clipboard button

### 3.4 Export (Flat Schema Only)

When the active template's schema is flat:
- Show a **Download CSV** button
- Show a **Download XLSX** button (optional, v2)

When schema is nested, export options are hidden with a tooltip: *"Export is available for flat schemas only."*

## 4. Architecture

```
┌─────────────────────────────────┐
│          Electron Shell         │
│  ┌───────────────────────────┐  │
│  │     React UI (renderer)   │  │
│  │  - Template Manager       │  │
│  │  - Extraction View        │  │
│  │  - Result Viewer          │  │
│  └────────────┬──────────────┘  │
│               │ HTTP(localhost) │
└───────────────┼─────────────────┘
                │
┌───────────────▼─────────────────┐
│       Python Backend Service    │
│  ┌───────────────────────────┐  │
│  │  POST /extract            │  │
│  │  - Receives: PDF + schema │  │
│  │  - Builds Pydantic model  │  │
│  │  - Calls doc-intelligence │  │
│  │  - Returns: JSON result   │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

The Python backend is started as a child process by Electron on app launch and killed on exit.

## 5. API Contract (Frontend ↔ Backend)

### `POST /extract`

**Request**
```json
{
  "pdf_path": "/absolute/path/to/file.pdf",
  "schema": {
    "invoice_number": { "type": "string" },
    "total_amount": { "type": "number" }
  },
  "extraction_config": {
    "extraction_mode": "single_pass",
    "include_citations": false,
    "page_numbers": null
  }
}
```

**Response**
```json
{
  "extracted_data": {
    "invoice_number": "INV-2024-001",
    "total_amount": 4500.00
  },
  "metadata": { },
  "is_flat": true
}
```

**Errors:** Standard HTTP status codes with `{ "error": "message" }` body.

---

## 6. Data Storage (Local)

Templates stored in a local JSON file or SQLite DB at a user-data path managed by Electron.

**Template record:**
```json
{
  "id": "uuid",
  "name": "Invoice Extractor",
  "description": "",
  "schema": { ... },
  "extraction_config": { ... },
  "created_at": "ISO8601",
  "updated_at": "ISO8601"
}
```

No cloud sync in v1.

---

## 7. Configuration

User-level config (stored locally):
- `openai_api_key` — required, set on first launch via a settings screen
- `default_model` — defaults to `gpt-4o-mini`
- `python_path` — auto-detected, overridable

---

## 8. Out of Scope (v1)

- Multi-file batch extraction
- Extraction history / job log persistence
- Cloud sync or user accounts
- Scanned PDF OCR (depends on `doc-intelligence` roadmap)
- Custom LLM providers beyond OpenAI
